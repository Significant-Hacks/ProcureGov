# ProcureGov Deployment Guide

## Prerequisites
- XAMPP (MySQL + Tomcat) installed and running
- NetBeans IDE with Java EE support
- Java JDK 8+

## Deployment Steps

### 1. Database Setup
The application requires a MySQL database named `DavidMohale2333908`. The schema file creates the database, all tables, and seed data in a single run.

**Via phpMyAdmin:**
1. Open `http://localhost/phpmyadmin`
2. Go to the **Import** tab
3. Select `sql/schema.sql` from the project
4. Click **Go**

**Via command line:**
```
mysql -u root < sql/schema.sql
```

### 2. Database Connection
The application connects to MySQL through a JNDI DataSource defined in `web/META-INF/context.xml`. Tomcat creates a connection pool on startup using these settings:

- **JNDI Name**: `jdbc/ProcureGovDB` â€” referenced by `DBConnectionUtil.java` at runtime
- **Driver**: `com.mysql.cj.jdbc.Driver` â€” provided by `mysql-connector-j-9.3.0.jar` in `WEB-INF/lib`
- **URL**: `jdbc:mysql://localhost:3306/DavidMohale2333908` â€” points to the database created in step 1
- **Pool**: 20 max active connections, 10 max idle, 10 second wait timeout
- **Credentials**: `root` / empty password (default XAMPP setup)

If your MySQL uses a different username or password, update the `username` and `password` attributes in `context.xml`.

### 3. Email System
The application sends email notifications (bid confirmations, award notices, password resets) via SMTP. Configuration is split across two files:

**`web/WEB-INF/web.xml`** â€” defines SMTP parameters as context-params:
- `smtpHost`: `smtp.gmail.com`
- `smtpPort`: `587`
- `smtpUser`: sender Gmail address
- `smtpPassword`: Gmail App Password (16-character)

**`web/META-INF/context.xml`** â€” declares a JNDI Mail Session resource named `mail/ProcureGovMail` for container-managed email.

The `MailConfigUtil` class reads the context-params from `web.xml` at runtime. To use a different email provider, change the `smtpHost`, `smtpPort`, `smtpUser`, and `smtpPassword` values in `web.xml`.

## File Upload System
Uploaded files (tender documents, bid documents, award confirmations) are stored on the server filesystem:

- **Location**: `${user.home}/ProcureGov/uploads/` â€” resolved at runtime using `System.getProperty("user.home")`
- **Auto-created**: The `DBInitListener` creates this directory on application startup
- **Cross-platform**: Works on Windows, macOS, and Linux without configuration changes
- **Configured in**: `web.xml` context-param `uploadDirectory` and `FileUtil.java`

## Troubleshooting

### Database Connection Fails
- Verify MySQL is running in XAMPP Control Panel
- Verify the database `DavidMohale2333908` exists (check phpMyAdmin)
- Verify `context.xml` credentials match your MySQL setup
- Verify port 3306 is not occupied by another service

### File Upload Fails
- Check that the application has write access to the user home directory
- The upload directory is created automatically on startup â€” check server logs if missing

### Email Not Sending
- Gmail requires an App Password (not the account password) â€” generate one at Google Account â†’ Security â†’ App Passwords
- Verify `smtpUser` and `smtpPassword` in `web.xml` are correct
- Check firewall/antivirus is not blocking outbound port 587

### NetBeans Datasource Warning
- A design-time warning about `jdbc/ProcureGovDB` is normal â€” the JNDI name is resolved at runtime by Tomcat, not by the IDE
- The `setup/sun-resources.xml` file helps NetBeans recognize the datasource reference
