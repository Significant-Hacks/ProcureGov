## Key Design Patterns & Architecture

### MVC Pattern Implementation
- **Model**: JavaBeans in `model/` package
- **View**: JSP pages in `web/pages/` organized by role
- **Controller**: Servlets in `controller/` package

### Security Architecture
- **Authentication**: SHA-256 password hashing with session management
- **Authorization**: Role-based access control via `AuthRedirectFilter`
- **Session Management**: Centralized in `SessionUtil`

### Data Persistence
- **Connection Pooling**: JNDI DataSource configured in `context.xml`
- **DAO Pattern**: Interface-based data access with JDBC implementations
- **Transaction Management**: Handled at DAO layer

### File Management
- **Upload Handling**: Part API in Servlets with server-side storage
- **Download Security**: Dedicated `DownloadServlet` prevents path exposure
- **Storage Location**: Relative path using `user.home/ProcureGov/uploads/` - no absolute paths
- **Utility**: `FileUtil` handles cross-platform file operations

### Email Integration
- **JavaMail API**: For supplier notifications (Module 6)
- **Configuration**: JNDI mail session in `context.xml` - no properties files
- **Utility**: `MailConfigUtil` handles JNDI lookups

## Tender Lifecycle Flow

```
Draft → Open → Closed → Under Evaluation → Evaluated → Awarded
  ↓        ↓        ↓           ↓              ↓         ↓
Officer  System   Evaluator   Officer        System    Officer
creates  publishes evaluates  triggers       calculates selects
tender   tender   bids        evaluation     scores    winner
```

### Status Enforcement
- **Server-side validation** in Servlets (no client-side transitions)
- **Automatic transitions** for system-triggered states
- **Manual transitions** for officer-controlled states
- **Role-based access** to different lifecycle stages
